◆ cia402_init()

void cia402_init ( co_dev_t *  dev)

Initialises the CiA 402 slave module.

Must be called once after canopen_init(). Clears all registers, sets the initial Statusword to CIA402_SW_ON_DISABLED, and writes it to the OD.

Parameters
[in]devPointer to the CANopen device object (slave OD).

◆ cia402_on_sync()

void cia402_on_sync ( uint8_t  cnt)

Main FSM tick – called on every SYNC frame.

Reads the current Controlword from the OD, advances the CiA 402 state machine, updates the Statusword and actual position in the OD. If sim_fault is set the FSM transitions to the Fault state regardless of the Controlword.

Parameters
[in]cntSYNC counter (0–240) forwarded by the CANopen stack.

void cia402_on_running ( void  )

Notifies the CiA 402 layer that the slave has entered OPERATIONAL.

Called from canopen.cfg.on_running. Resets the FSM to the initial Switch-On Disabled state so the master can start a fresh enable sequence.

◆ cia402_on_stopped()

void cia402_on_stopped ( void  )

Notifies the CiA 402 layer that the slave has left OPERATIONAL.

Called from canopen.cfg.on_stopped (NMT STOP, HB timeout, bus error). Issues a Quick Stop and resets the Statusword to Switch-On Disabled.

Called after each TPDO transmission.

◆ cia402_master_on_running()

void cia402_master_on_running ( void  )

Notifies the FSM that the slave has entered the OPERATIONAL state.

Called from canopen.cfg.on_running when st_ind() reports CO_NMT_ST_START for the monitored node. Starts the drive enable sequence: SDO_INIT → SHUTDOWN → SWITCH_ON → ENABLE_OP → RUNNING.

◆ cia402_master_on_stopped()

void cia402_master_on_stopped ( void  )

Notifies the FSM that the slave has left the OPERATIONAL state.

Called from canopen.cfg.on_stopped on HB timeout, BOOTUP, or NMT STOPPED. Issues a Quick Stop Controlword and returns to CIA402_M_IDLE so the sequence restarts when the slave recovers.

◆ cia402_master_on_sync()

void cia402_master_on_sync ( uint8_t  cnt)

Main FSM tick – called on every SYNC frame.

Advances the FSM based on the current drive Statusword. In CIA402_M_RUNNING state, this is the application point where the target position is written to the OD and sent via TPDO.

Parameters
[in]cntSYNC counter (0–240) forwarded by the CANopen stack.

void cia402_master_set_target ( int32_t  pos)

Sets the desired target position from outside the module.

Safe to call from an interrupt or another module. The value is forwarded to the drive immediately only when the FSM is in CIA402_M_RUNNING; otherwise it is stored and applied on the next RUNNING entry.

Parameters
[in]posDesired target position [pulses].
