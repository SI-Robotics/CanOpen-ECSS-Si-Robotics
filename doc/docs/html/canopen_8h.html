Detailed Description

CANopen slave layer – initialisation, processing and configuration.

Wraps the Lely CANopen stack (can_net, co_nmt, PDO objects) for a slave node and exposes a simple application interface through canopen_config and the canopen_init / canopen_process functions.

Supports dual-bus redundancy (Bus A / Bus B) via the ECSS redundancy object at OD index 0x2000.

◆ canopen_init()

canopen_status canopen_init ( canopen_config  conf)

Initialises the CANopen slave stack.

Allocates and configures: memory pool, can_net, Object Dictionary, NMT, RPDO and TPDO objects. Registers all internal stack callbacks and writes the ECSS redundancy parameters (OD 0x2000) to the OD. Ends by issuing NMT RESET_NODE, transitioning the slave to Pre-Operational and then to Operational once the master issues NMT START.

Parameters
[in]confLayer configuration (HAL functions + application callbacks).
Returns
CANOPEN_OK on success, CANOPEN_ERROR on any failure.

◆ canopen_process()

void canopen_process ( void  )

Processes all pending CANopen events – call from the main loop.

Responsibilities (in order):

  1. Advances the Lely stack clock to the current monotonic time.
  2. Drains all pending CAN frames from bus A then bus B.
  3. Executes a deferred communication reset if reset_comm_pending is set.
Note
Not thread-safe; call only from a single execution context.

Responsibilities (in order):

  1. Updates the stack's internal clock via can_net_set_time().
  2. Drains all pending CAN frames from bus A and bus B.
  3. Handles a deferred bus switchover when slave_hb_timeout is set.
  4. Runs per-state logic for the current bus_state.
Note
Not thread-safe; must be called from a single execution context.
