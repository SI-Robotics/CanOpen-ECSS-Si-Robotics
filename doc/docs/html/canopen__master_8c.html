Detailed Description

CANopen master layer – implementation.

Contains the initialisation sequence, main processing loop, and all internal Lely stack callbacks (NMT state, Heartbeat, SYNC, RPDO, TPDO). The public API is declared in canopen_master.h.

Heartbeat consumer indication callback.

Registered via co_nmt_set_hb_ind(). Called by the Lely stack whenever the HB consumer (OD 0x1016) detects an event for a monitored node.

Behaviour:

  • CO_NMT_EC_TIMEOUT / CO_NMT_EC_OCCURRED : sets slave_hb_timeout and fires on_stopped so the main loop can trigger a bus switch on the next canopen_process() call.
  • CO_NMT_EC_TIMEOUT / resolved : resets bus_switch_count to 0.
  • CO_NMT_EC_STATE : logs the state change only.
Parameters
[in]nmt_NMT master object (forwarded to co_nmt_on_hb()).
[in]idNode-ID of the node that triggered the event.
[in]ec_stateWhether the event occurred or was resolved.
[in]reasonReason: timeout or state change.
[in]dataUser data pointer (unused).

static

NMT state change indication callback.

Registered via co_nmt_set_st_ind(). Called by the Lely stack when a monitored node broadcasts a new NMT state. Only events from CANOPEN_REMOTE_NODE are processed; all others are silently ignored.

State handling:

  • CO_NMT_ST_BOOTUP : records boot time, detects duplicate Node-IDs (two BOOTUPs within 2 s → system locked), fires on_stopped.
  • CO_NMT_ST_STOP : transitions to CANOPEN_BUS_PREOP, fires on_stopped.
  • CO_NMT_ST_START : if not locked, transitions to CANOPEN_BUS_OP and fires on_running.
  • CO_NMT_ST_PREOP : if not locked, sends NMT START to the slave.
Parameters
[in]nmt_NMT master object (used to issue NMT CS requests).
[in]idNode-ID of the node whose state changed.
[in]stNew NMT state (CO_NMT_ST_* value).
[in]dataUser data pointer (unused).

Switches the active CAN bus between A and B.

Increments bus_switch_count and calls co_nmt_set_active_bus(). If the maximum number of switches (CANOPEN_MAX_BUS_SWITCHES) is reached, enter_error() is called instead. On success the master transitions to CANOPEN_BUS_PREOP and waits for the slave to re-announce itself via Heartbeat or BOOTUP.

◆ canopen_process()

void canopen_process ( void  )

Processes all pending CANopen events – call from the main loop.

Responsibilities (in order):

  1. Updates the stack's internal clock via can_net_set_time().
  2. Drains all pending CAN frames from bus A and bus B.
  3. Handles a deferred bus switchover when slave_hb_timeout is set.
  4. Runs per-state logic for the current bus_state.
Note
Not thread-safe; must be called from a single execution context.

◆ duplicate_node_locked

bool duplicate_node_locked = false
static

Duplicate Node-ID guard flag.

Set to true when two BOOTUP frames from the same node arrive within 2000 ms. Once locked the master ignores all subsequent NMT START and PRE-OP events from that node to prevent uncontrolled drive enable.
